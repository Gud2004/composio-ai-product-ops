Executive Summary

This project automates the research of SaaS applications by collecting structured information about authentication methods, API availability, self-service onboarding, Model Context Protocol (MCP) support, developer experience, and integration feasibility.

The solution is designed as a reusable research pipeline that reads application names from a CSV file, queries an LLM, validates structured JSON output, stores individual reports, and prepares data for further analysis and verification.

The pipeline supports resumable execution, allowing interrupted runs to continue without repeating completed research.

Assignment Goals

  • Research 100 SaaS applications
  • Collect structured metadata
  • Generate machine-readable JSON
  • Support resumable execution
  • Analyze ecosystem-wide patterns
  • Verify findings against official documentation

Research Methodology

Stage Description
Input Read 100 SaaS applications from apps.csv
Research Query Gemini API with a structured research prompt
Validation Ensure output follows the required JSON schema
Storage Save one JSON file per application
Analysis Aggregate authentication methods, API types, buildability and blockers
Verification Cross-check a sample of applications using official documentation

Key Findings

  • OAuth 2.0 was the dominant authentication mechanism across modern SaaS platforms.
  • REST APIs were significantly more common than GraphQL APIs.
  • Most developer-focused SaaS products offered self-service onboarding.
  • Enterprise platforms frequently required administrator approval before integration.
  • API rate limiting was the most common integration challenge.
  • MCP support is still emerging and available for only a limited number of platforms.

Verification Strategy

To improve reliability, the generated research was compared against official developer documentation for a representative sample of applications. The verification focused on:

  • Authentication mechanisms
  • API availability
  • Developer onboarding process
  • MCP availability
  • Official documentation links

The pipeline was designed to support iterative verification and correction whenever discrepancies were identified.

Limitations

  • LLM responses depend on publicly available documentation.
  • Enterprise-only APIs may require authenticated developer accounts.
  • Free-tier Gemini API quotas limited uninterrupted execution during large-scale runs.
  • The pipeline supports resumable execution and batching to continue processing after quota resets.
